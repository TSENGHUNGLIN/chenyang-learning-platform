## 🤖 三、AI分析整合方案

### 1. AI分析出題流程
```
上傳培訓教材/專案文件
    ↓
AI分析內容
    ↓
自動生成題目
    ↓
記錄「考題出處」（檔案名稱、章節、頁碼）
    ↓
自動標記「AI生成」標籤
    ↓
建議分類和標籤
    ↓
人工審核確認
    ↓
匯入題庫
```

### 2. 考題出處追蹤欄位
- **出處類型**：AI生成 / 人工編寫
- **來源檔案**：原始文件名稱
- **章節位置**：章節、頁碼
- **生成時間**：AI分析日期
- **審核狀態**：待審核 / 已審核 / 已修改

### 3. AI分析建議分類邏輯
根據文件內容關鍵字自動建議：
- **設計類關鍵字** → 建議「設計美學」分類
- **工程類關鍵字** → 建議「工程管理」分類
- **客戶類關鍵字** → 建議「客戶關係」分類
- **管理類關鍵字** → 建議「領導管理」分類

## 📊 四、實際應用範例

### 範例一：新人設計師轉正考試
**篩選條件**：
- 分類：設計美學 > 設計基礎
- 標籤：基礎 + 設計助理 + 轉正考核 + 必考
- 結果：精準找到適合新人轉正的基礎設計題目

### 範例二：資深設計師晉升考試
**篩選條件**：
- 分類：領導管理 > 團隊領導
- 標籤：進階 + 資深設計師 + 晉升考核 + 專案管理
- 結果：找到適合晉升主管的領導力題目

### 範例三：工程監造專業認證
**篩選條件**：
- 分類：工程管理 > 施工技術
- 標籤：專家 + 工程監造 + 專業認證 + 工程知識
- 結果：找到工程監造專業認證題目

## 🎯 五、實施計畫

### 第一階段：建立基礎架構
1. 批次建立8大主分類
2. 建立所有子分類（約50個）
3. 建立8類標籤體系（約60個標籤）

### 第二階段：AI功能整合
1. 修改AI分析功能，新增「考題出處」記錄
2. 自動標記「AI生成」標籤
3. 實作分類和標籤建議功能

### 第三階段：批次匯入現有題目
1. 為現有題目設定分類
2. 批次標記標籤
3. 補充考題出處資訊

### 第四階段：使用者培訓
1. 建立使用手冊
2. 培訓管理員和編輯者
3. 建立最佳實踐指南
